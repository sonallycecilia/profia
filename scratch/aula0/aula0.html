Aula 0 - Variáveis de Instância

Variáveis de Instância

Para a aula de hoje, vamos mergulhar no mundo da organização do Scratch! Entender variáveis é o que separa um jogo simples de um jogo profissional e inteligente.

O que é uma variável?

Imagine que você precisa marcar quantos pontos fez ou quanto tempo falta para acabar a fase. Como o computador "lembra" disso? Ele usa uma Variável. Uma variável é como uma caixa onde guardamos uma informação. Nós damos um nome para essa caixa (como "Pontos") e podemos mudar o que está dentro dela a qualquer momento.

Tipos de Informação (O que cabe na caixa?)

  • Números (Inteiros e Decimais)

    Usados para contar pontos, vidas, tempo ou definir a posição (x, y) de um ator. Exemplo: Pontos = 10

  • Texto (Strings)

    Utilizado para guardar palavras, nomes de personagens ou mensagens de diálogo. Exemplo: Nome = "Gato"

  • Variáveis Globais

    Criadas para "Todos os atores". Funcionam como o placar do jogo: todo mundo vê o mesmo valor.

  • Variáveis de Instância

    Criadas "Apenas para este ator". Cada clone ganha sua própria caixa exclusiva.

Visibilidade da Variável

Global (Para todos)

Informações gerais que afetam todos ao mesmo tempo.

Exemplos: Placar, Recorde, Cronômetro.
Instância (Privada)

Características individuais de cada clone ou ator.

Exemplos: Vida de cada inimigo, Velocidade, IDs.

Conceitos de Aprofundamento

  • Escopo: É o "alcance" da variável. Define quem pode ver ou mexer na informação.
  • Clonagem: Cada clone nasce com sua própria cópia da variável de instância, independente do original.
  • Encapsulamento: É proteger a informação. Um ator não consegue "bagunçar" a variável de outro.

Vamos explorar? Mãos na massa!

Neste experimento, vamos criar vários clones de um robô para testar a diferença visual entre os dois tipos de variáveis.

Parte 1: O Erro (Variável Global)

Se usarmos "Para todos os atores", todos os robôs compartilham o mesmo "coração".

CÓDIGO DO ERRO
Quando eu começar como um clone
 ➜ Sempre
    ➜ Se tocado por Ponteiro do Mouse então
       ➜ Variável: Adicione -1 a Vida_Global
    ➜ Se Vida_Global < 1 então
       ➜ Controle: Apague este clone

O que acontece: Ao tocar em UM robô, a vida de TODOS diminui e eles explodem juntos!

Parte 2: A Solução (Variável de Instância)

Agora, vamos dar a cada robô sua própria memória individual marcando "Apenas para este ator".

  • Configuração: No ator principal, use esconda e mude Minha_Vida para 3.

  • Vida Individual: No bloco do clone, garanta que ele use mostre e receba sua própria vida.

CÓDIGO DA SOLUÇÃO
Quando eu começar como um clone
 ➜ Aparência: Mostre
 ➜ Variável: Mude Minha_Vida para 3
 ➜ Sempre
    ➜ Se tocado por Ponteiro do Mouse então
       ➜ Variável: Adicione -1 a Minha_Vida
    ➜ Se Minha_Vida < 1 então
       ➜ Controle: Apague este clone
Dica de Mestre:
Variável de Instância = Individualidade. Use-a sempre que quiser que seus clones tenham comportamentos ou estados diferentes uns dos outros!
